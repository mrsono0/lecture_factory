# 강사대본 생성 (Phase 8) 세부 워크플로우

### 설계 원칙

| 차원 | 교안 Phase 6 | 대본 Phase 8 |
|------|-------------|-------------|
| 산출물 성격 | 학습 콘텐츠 (What) | 전달 스크립트 (How) — 강사가 바로 수업 가능 |
| 상세도 | 교시당 개념설명+다수 예시 | 교시당 발화문+발문+시간큐+전환 |
| 고유 요소 | 다중 예시(★/◇), 완성 코드, 오개념 교정 | 발화문, 발문 스크립트, 행동지시, 시간큐, 전환문구 |
| 참조 관계 | 독립 | 교안 참조 기반 — 교안에 없는 팩트 창작 금지 |

**5대 핵심 원칙**:

1. **교안 참조 기반**: 교안의 설명을 발화문으로 변환. 교안에 없는 팩트 창작 금지.
   - 코드 시연 시: 교안의 코드를 그대로 인용 (대본에서 코드 새로 작성 금지)
   - 교안 참조: `[교안 §I Do ★ 예시 1 사용]`, `[교안 §오개념 참조]` 형식

2. **GRR + 3단 구성**: 도입/전개/정리 구분 + GRR(I Do/We Do/You Do) 구간 구분
   - 도입: 주의 획득 → 학습목표 안내 → 선수학습 확인
   - 전개: I Do(시범) → We Do(안내) → You Do(독립)
   - 정리: Exit 평가 → 핵심 요약 → 다음 차시 예고

3. **시간 관리**: 50분에 맞춰 교안 콘텐츠를 선택적 배치
   - ★ 핵심 예시: 반드시 포함
   - ◇ 심화 예시: `[시간 여유 시]` 태그로 선택 배치
   - 시간 표기: `(경과: N분 / 잔여: N분)`

4. **구어체 발화**: 친절한 구어체 (~해요, ~입니다). 딱딱한 문어체(~한다) 금지.
   - 전문 용어 사용 시 즉시 일상 언어로 부연
   - input_data.json의 tone(비유 중심 등) 반영

5. **대본 전용 표기**: 발화문과 행동지시만 사용
   - `> "..."`: 강사 발화문
   - `❓`: 발문 (Bloom's `[Ln]` 선택적 표기)
   - `💬`: 예상 학습자 반응 / 오답 대응
   - `[...]`: 행동 지시 (비발화 동작)
   - 나머지 이모지(📋, ⏱️, 🔄) 사용하지 않음 — 텍스트로 표기

### Step 1: 차시별 대본 작성

각 session_D*.md에 대응하는 narration_D*.md를 작성한다.

#### 1-1. 도입부 대본 작성

교안 §도입의 내용을 발화문으로 변환:
- Hook/동기유발 → 스토리텔링 발화문
- 학습목표 → "이 시간이 끝나면 ~할 수 있게 됩니다" 패턴
- 선수학습 확인 → 복습 발문 + 예상 반응
- 형성평가(도입) → 교안 참조 링크

#### 1-2. I Do 대본 작성

교안 §I Do의 내용을 발화문으로 변환:
- 핵심 개념 설명 → 구어체 발화문 (비유 포함)
- ★ 예시 → `[코드 시연: 교안 ★ 예시 N]` + 코드 설명 발화
- ◇ 예시 → `[시간 여유 시: 교안 ◇ 예시 N]` 태그로 선택 배치
- 오개념 교정 → 교정 발문 + 발화
- 발문: brainstorm §1에서 해당 차시 전개부 발문 선택

#### 1-3. We Do 대본 작성

교안 §We Do의 내용을 발화문으로 변환:
- 스캐폴드 코드 → `[교안 §We Do 스캐폴드 코드 시연]` + 단계별 안내 발화
- 예상 실수 → 대응 발화
- 형성평가(전개) → 교안 참조

#### 1-4. You Do 대본 작성

교안 §You Do의 내용을 발화문으로 변환:
- 과제 안내 → 발화문
- 순회 피드백 → 흔한 실수 발견 시 발화 예시
- 시작 코드 → 교안 참조

#### 1-5. 정리부 대본 작성

교안 §정리의 내용을 발화문으로 변환:
- 형성평가(Exit) → 안내 발화 + 교안 참조
- 핵심 요약 → "오늘 배운 핵심 세 가지: ①②③" 패턴
- 실무 적용 → 전이 발화
- 다음 차시 → 예고 발화


### Overlap 컨텍스트 수신 규칙

교안 Phase 6과 동일. 이전 블록의 마지막 narration 파일 정리 섹션 참조.

### revision 모드

Phase 9 대본 검토에서 REVISION_REQUIRED 시:
1. `_review_narration_{block_id}.md` Read → 위반 사항 추출
2. 해당 차시의 narration_D*.md만 재작성
3. 교안 session_D*.md는 절대 수정하지 않음

### 금지 사항

**공통**: `shared/prohibited-rules.md`를 Read하여 따른다.

**대본 추가**:
- **교안 콘텐츠 변경 금지**: session_D*.md의 내용을 수정하지 않는다
- **팩트 창작 금지**: 교안에 없는 사실, 코드, 예시를 대본에서 새로 만들지 않는다
- **코드 신규 작성 금지**: 코드는 교안에서 참조만 한다. 대본에서 새 코드 작성 금지
- **교안 참조 없는 시연 금지**: `[코드 시연]`에는 반드시 교안 예시 번호를 명시
