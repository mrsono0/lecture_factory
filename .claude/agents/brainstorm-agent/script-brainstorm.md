# 강의교안 브레인스토밍 (Phase 3) 세부 워크플로우

구성안이 **"무엇을 가르칠까(What)"**를 정의했다면, 교안 Phase 3은 **"어떻게 가르칠까(How)"**를 브레인스토밍한다.
발문 설계, 학습활동 구상, 실생활 사례 도출, Gagne 9사태 구현 방안이 핵심 산출물이다.

### 전체 흐름

```
Step 0: 입력 로드 + 브레인스토밍 계획 수립
  │     input_data.json + research_exploration.md + 구성안 → brainstorm_plan.md
  │
  ├── Step 1: 발산적 아이디어 생성 → divergent_ideas.md
  │   (4가지 발산 기법 × 리서치 인사이트 시드 × 4가지 카테고리)
  │
  ├── Step 2: 차시별 매핑 + 클러스터링 → idea_clusters.md
  │   (차시 귀속, 수업 단계, GRR 단계, Gagne 사태 기준)
  │
  ├── Step 3: 다관점 검증 → review_result.md
  │   (3개 역할 순차 시뮬레이션 + Decision Log)
  │
  ├── Step 4: Bloom's 발문 설계 + Gagne 매핑 + 활동 구체화
  │   (검증 통과 아이디어를 구체적 발문/활동/Gagne 구현으로 정교화)
  │
  └── Step 5: 통합 산출물 작성 → brainstorm_result.md ★
      (8개 섹션 구조화 문서)
```

### 산출물 목록

```
{output_dir}/
├── brainstorm_plan.md      # Step 0: 브레인스토밍 계획
├── divergent_ideas.md      # Step 1: 발산 아이디어 원시 목록
├── idea_clusters.md        # Step 2: 차시별 클러스터링
├── review_result.md        # Step 3: 다관점 검증 + Decision Log
└── brainstorm_result.md    # Step 5: 최종 통합 산출물 ★
```

### Step 0: 입력 로드 + 브레인스토밍 계획 수립

| 항목 | 내용 |
|------|------|
| 입력 | `{output_dir}/input_data.json`, `{output_dir}/research_exploration.md`, `{source_outline.outline_path}`, `{source_outline.architecture_path}` |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/brainstorm_plan.md` |

**동작**:

1. `.claude/templates/input-schema-script.json` 읽기 — script_config 각 필드의 enum 값, 의미, 필드 간 관계를 사전 이해
2. `input_data.json` 읽기 — 교안 특화 필드 추출:
   - 구성안 필드: `topic`, `target_learner`, `learning_goals`, `keywords`, `lab_environment` (Q14)
   - `script_config`: `teaching_model`, `activity_strategies`, `time_ratio`, `bloom_question_map`, `instructional_model_map`, `formative_assessment`
3. `research_exploration.md` §7 리서치 인사이트 추출 (브레인스토밍 시드):
   - 각 인사이트의 SLO/activity_strategy/Bloom's 태깅 확인
   - 방향성만 시드로 활용 — 구체적 교안·대본에 고착되지 않도록 주의
4. `lecture_outline.md` 읽기 — 차시 구조, SLO 목록, 하위 주제 목록 추출
5. `architecture.md` (구성안) 읽기 — Gagne 적용 계획, GRR 패턴, 형성평가 배치
6. 교수 모델별 브레인스토밍 초점 설정:
   - `direct_instruction`: "어떻게 모델링·비계·점진적 해제를 설계할 것인가?"
   - `pbl`: "어떤 문제 시나리오가 학습을 촉발하는가?"
   - `flipped`: "사전학습-교실활동의 역할 분담은?"
   - `mixed`: 차시별 위 3개 조합
7. 차시 수 × 수업 단계(도입/전개/정리) = 브레인스토밍 단위 수 산출
8. 적용할 발산 기법 선택 (3~4개)
9. `brainstorm_plan.md` 작성:
   - 브레인스토밍 시드 목록 (리서치 인사이트 + SLO 매핑)
   - 선택된 발산 기법과 선택 이유
   - 교수 모델별 초점 질문
   - 검증 관점 정의
   - 브레인스토밍 단위 수 (차시 × 수업 단계)

### Step 1: 발산적 아이디어 생성 (교안 특화 4가지 기법)

| 항목 | 내용 |
|------|------|
| 입력 | `brainstorm_plan.md`, `input_data.json`, `research_exploration.md`, 구성안 파일 |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/divergent_ideas.md` |

#### 교안 특화 4가지 발산 기법

| # | 기법 | 적용 방법 | 생성 대상 |
|---|------|----------|----------|
| 1 | **HMW(How Might We) 재구성** | 각 SLO를 "어떻게 하면 학습자가 ~할 수 있을까?" HMW 질문으로 변환. 배달 형태(강의/실습)에 갇히지 않고 학습 문제 자체에 집중 | 학습활동, 설명 전략 |
| 2 | **학제간 융합** | 인접 분야의 교수법/활동 패턴을 차용 (예: 게이미피케이션+코딩교육, 의학교육 시뮬레이션→소프트웨어 실습) | 학습활동, 실생활 사례 |
| 3 | **SCAMPER 활동 설계** | 기존 활동 유형(강의/실습/토론/프로젝트)에 대체/결합/적응/변형/전용/제거/역전 적용. 교안 활동 다양화에 특화 | 학습활동, 형성평가 |
| 4 | **강제 제약 + Magic Wand** | 극단적 제약 부과("슬라이드 없이 5분 설명?") 후 현실화. 반대로 제약 완전 해제("완벽한 교안이라면?") 후 점진적 축소 | 발문, 도입부 훅, 설명 전략 |

#### SCAMPER 교안 활동 설계 버전

| SCAMPER | 교안 적용 질문 |
|---------|--------------|
| **S**ubstitute | "이 개념의 시범을 다른 매체/도구로 대체하면?" |
| **C**ombine | "이론 설명과 실습을 하나의 활동으로 결합하면?" |
| **A**dapt | "다른 분야에서 이 유형의 실습을 어떻게 진행하는지 차용하면?" |
| **M**odify | "난이도를 조절하면? GRR 단계를 늘이거나 줄이면?" |
| **P**ut to another use | "이 실습을 형성평가로도 활용하면?" |
| **E**liminate | "이 단계 없이도 학습자가 수행할 수 있는가?" |
| **R**everse | "결과부터 보여주고 과정을 역추적하면? You Do → I Do 역전?" |

#### 동작

1. 리서치 인사이트 각각을 시드로, 선택된 발산 기법 3~4개를 순차 적용
2. **4가지 카테고리별** 아이디어 생성 (구성안의 "하위 주제" 대신):
   - **발문 아이디어**: Bloom's 6수준 × 수업 단계(도입/전개/정리) 조합
   - **학습활동 아이디어**: activity_strategies × GRR 단계(I Do/We Do/You Do) 조합
   - **실생활 사례/훅**: 도입부 주의 획득 + 전개부 실무 연결 + 정리부 전이 촉진
   - **설명 전략/비유**: 핵심 개념별 비유, 예시-비예시 쌍, 시각화 방법
3. 각 아이디어 기록 형식:
   ```
   ### 아이디어 {번호}: {제목}
   - 카테고리: {발문/활동/사례/설명전략}
   - 설명: {1~2문장}
   - 발산 기법: {사용한 기법명}
   - 시드: {관련 리서치 인사이트 번호}
   - 관련 SLO: {SLO1-1, ...}
   - 관련 차시: {D1-3, D2-1, ...}
   - Bloom's 수준: {L1~L6}
   ```
4. 의도적 과잉 생성: 최종 필요량의 2~3배 (후속 Step에서 선별)
5. 평가 유보: 이 단계에서는 품질 판단을 하지 않음
6. `divergent_ideas.md` 작성

### Step 2: 차시별 매핑 + 클러스터링

| 항목 | 내용 |
|------|------|
| 입력 | `divergent_ideas.md`, `input_data.json`, 구성안 `architecture.md` |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/idea_clusters.md` |

#### 클러스터링 기준 (교안 특화 4가지)

- **차시 귀속**: 어떤 Day/차시에 적용할 수 있는 아이디어인가
- **수업 단계 귀속**: 도입/전개(초반·후반)/정리 중 어디에 적합한가
- **GRR 단계 귀속**: I Do/We Do/You Do Together/You Do Alone
- **Gagne 사태 귀속**: 9사태 중 어느 사태에 해당하는가

#### 동작

1. 발산 아이디어를 차시별 × 수업 단계별로 매핑
2. 클러스터 간 관계 식별 (연쇄 활동, 반복 패턴, 난이도 점진)
3. 학습자 페르소나 활용 — 구성안 brainstorm_result.md의 페르소나 재사용 (신규 생성 불필요)
4. 클러스터 요약 테이블:
   ```
   | 차시 | 수업단계 | GRR | Gagne사태 | 발문 아이디어 | 활동 아이디어 | 사례/훅 |
   |------|---------|-----|----------|------------|------------|--------|
   ```
5. `idea_clusters.md` 작성

### Step 3: 다관점 검증 (3개 역할)

| 항목 | 내용 |
|------|------|
| 입력 | `idea_clusters.md`, `input_data.json` |
| 도구 | Read, Write |
| 산출물 | `{output_dir}/review_result.md` |

구성안의 5개 역할에서 교안 특화 3개로 축소한다.
구성안에서 이미 하위 주제·SLO·구조가 확정되어 있으므로, 교안 단계에서는 "구조 변경"이 아닌 **"실행 세부사항의 현실성 검증"**만 필요하다.
단일 에이전트가 순차적으로 각 관점을 시뮬레이션한다 (Agent 중첩 금지 제약).

#### 교안 도메인 역할 정의

**1. 비판적 교육자 (Critical Educator)**

| 항목 | 내용 |
|------|------|
| 역할 | SLO-활동 정렬, Bloom's 수준 적합성, Gagne 사태 누락 식별 |
| 핵심 질문 | "이 활동이 정말 해당 SLO 달성에 기여하는가?", "Bloom's 수준이 의도와 맞는가?", "Gagne 9사태 중 빠진 것은?" |
| 금지 | 새로운 아이디어 제안 금지 — 기존 아이디어의 적합성만 판단 |

**2. 학습자 대변인 (Learner Advocate)**

| 항목 | 내용 |
|------|------|
| 역할 | 인지 부하, 동기 유발, 난이도 적합성 검증 |
| 핵심 질문 | "학습자가 이 활동에서 왜 해야 하는지 이해하는가?", "인지 부하가 과도한 부분은?", "동기가 유지되는가?" |
| 금지 | 교수법 변경 제안 금지 — 학습자 경험 관점에서만 평가 |

**3. 시간 관리자 (Time Guardian)**

| 항목 | 내용 |
|------|------|
| 역할 | 차시별 시간 현실성, 활동 소요 시간, 도입/전개/정리 비율 검증 |
| 핵심 질문 | "이 활동의 실제 소요 시간은?", "도입/전개/정리 비율(time_ratio)이 지켜지는가?", "이동·준비 시간이 반영되었는가?" |
| 금지 | SLO 변경 제안 금지 — 시간 현실성만 판단 |

#### 동작

1. 관점 1~3을 순차 시뮬레이션 — 각 관점별 섹션으로 기록
2. 마무리: 충돌 해결 + 판정 태그 부여
   - `ADOPT`: 그대로 채택
   - `REVISE`: 수정 후 채택 (수정 방향 명시)
   - `DROP`: 폐기 (근거 명시)
   - `MERGE`: 다른 아이디어와 병합
3. Decision Log 작성:
   ```
   | # | 판정 | 대상 아이디어 | 수정 방향/병합 대상 | 이유 |
   |---|------|------------|-------------------|------|
   ```
4. `review_result.md` 작성

### Step 4: Bloom's 발문 설계 + Gagne 9사태 매핑 + 활동 구체화

| 항목 | 내용 |
|------|------|
| 입력 | `review_result.md`, `input_data.json`, `brainstorm_plan.md` |
| 도구 | Read, Write |
| 산출물 | (Step 5에서 brainstorm_result.md에 통합) |

#### 4-1. Bloom's 기반 발문 설계

`bloom_question_map.per_session`의 차시별 수업 단계별 발문 수준(L1~L6)을 참조하여, Step 1~3에서 검증 통과한 발문 아이디어를 구체적 발문 문장으로 정교화한다.

**발문 유형 체계 (소크라테스식 6유형 + Bloom's 매핑)**:

| 발문 유형 | 교안 적용 시점 | Bloom's 수준 |
|----------|--------------|-------------|
| 명료화 질문 | 도입, 전개 초반 | L1~L2 |
| 가정 탐색 질문 | 전개 중반 | L3~L4 |
| 근거 탐구 질문 | 전개 중반~후반 | L3~L4 |
| 함의·결과 질문 | 전개 후반, 정리 | L4~L5 |
| 관점·시각 질문 | 전개 중반 | L4~L5 |
| 메타인지 질문 | 정리 | L5~L6 |

**차시별 발문 시퀀스**: 도입(선수 확인 L1~L2 → Hook) → 전개 초반(개념 확인 L2~L3) → 전개 후반(심화 분석 L4~L5) → 정리(통합·전이 L3~L6)

#### 4-2. Gagne 9사태 교안 구현 방안

각 차시의 수업 단계별로 Gagne 9사태의 구현 방안을 구체적으로 설계:

| Gagne 사태 | 수업 단계 | 구현 아이디어 카테고리 |
|-----------|----------|-------------------|
| 1. 주의 획득 | 도입 | 훅/사례/질문/놀라운 사실 |
| 2. 목표 고지 | 도입 | SLO의 학습자 언어 변환 |
| 3. 선수 학습 자극 | 도입 | 복습 발문/마인드맵/퀴즈 |
| 4. 자극 자료 제시 | 전개 | 멀티미디어/예시-비예시/단계별 데모 |
| 5. 학습 안내 | 전개 | 체크리스트/힌트카드/개념지도 |
| 6. 수행 유도 | 전개 | GRR 단계별 실습/과제 |
| 7. 피드백 제공 | 전개~정리 | 즉각/동료/루브릭 피드백 |
| 8. 수행 평가 | 정리 | 형성평가(assessment_plan 연동) |
| 9. 파지·전이 촉진 | 정리 | 실무 적용 과제/다른 맥락 탐색 |

#### 4-3. 학습활동 구체화

검증 통과한 활동 아이디어를 교수 모델별 GRR 구조에 따라 구체화:

| 교수 모델 | GRR 중심 | 활동 구체화 초점 |
|---------|---------|--------------|
| 직접교수법 | I Do→We Do→You Do 전면 | 시범/안내 실습/독립 실습의 전환 지점과 비계 설계 |
| PBL | You Do Together 중심 | 문제 시나리오, 그룹 탐구 프로토콜, 발표·성찰 구조 |
| 플립러닝 | We Do→You Do Together | 사전학습 확인 활동, 교실 내 심화 활동, 적용 과제 |

#### 4-4. 학습자 오개념 해소 전략

`research_exploration.md` §4에서 식별된 오개념·장벽을 차시별 개입 지점에 매핑:
- 어느 차시의 어느 시점에서 오개념이 나타날 가능성이 높은가
- 어떤 발문/활동으로 교정하는가 (예시-비예시 쌍, 반증 제시)
